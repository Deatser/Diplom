# Left 2 Solve — Design Document

**Дата:** 2026-05-26  
**Статус:** Approved  
**Цель:** Дипломная работа — 1-минутный геймплейный ролик

---

## 1. Обзор проекта

**Left 2 Solve** — кооперативный браузерный платформер для двух игроков. Два персонажа поднимаются на вершину горы через 10 уровней. Игра невозможна в одиночку — все головоломки требуют одновременных действий обоих игроков.

### Приоритеты дипломного MVP

| Компонент | Приоритет | Статус |
|---|---|---|
| Level 1 — полный (туториал + дэш) | 🔴 Критично | Полная реализация |
| Level 10 — полный (все механики) | 🔴 Критично | Полная реализация |
| Главное меню (Hollow Knight стиль) | 🔴 Критично | Полная реализация |
| Лобби + система серверов | 🔴 Критично | Полная реализация |
| Настройки (язык, звук, видео, клавиатура) | 🟡 Важно | Полная реализация |
| LAN мультиплеер (Socket.io) | 🔴 Критично | Полная реализация |
| Levels 2–9 | 🟢 Низко | Заглушки |
| Интернет-сервер (Railway/Render) | 🟢 Опционально | В конце, если время |

## 4. Управление

### Оба игрока (одинаковые биндинги по умолчанию)

| Действие | Клавиша |
|---|---|
| Движение влево/вправо | A / D |
| Прыжок / Взаимодействие | W или Space |
| Двойной прыжок (если открыт) | W×2 или Space×2 в воздухе |
| Дэш | Shift |
| Заряженный дэш (Lvl 10) | Удержать Shift → отпустить |
| Удар о землю / Рывок вниз | S (в воздухе) |
| Крюк-кошка / Призыв платформы / Обмен | ПКМ (контекстно — зависит от открытых способностей) |

> Все клавиши перебиндируются в Настройки → Клавиатура. Биндинги сохраняются в localStorage.

## 5. Десять способностей

| # | Уровень | Название | Триггер | Описание |
|---|---|---|---|---|
| 1 | Lvl 1 | **Дэш** | Shift | Горизонтальный рывок на 8× длины. Cooldown 0.5с. В воздухе — только до приземления. |
| 2 | Lvl 2 | **Двойной прыжок** | W/Space ×2 | Второй прыжок в воздухе. Сбрасывается при приземлении. |
| 3 | Lvl 3 | **Цепляние за стены** | A/D у стены | Медленное скольжение + прыжок от стены в противоположную сторону. |
| 4 | Lvl 4 | **Удар о землю** | S в воздухе | Резкое падение. При приземлении — ударная волна, активирует плиты в радиусе 3 тайлов. |
| 5 | Lvl 5 | **Воздушный рывок вниз** | Shift+S в воздухе | Диагональный рывок вниз-вперёд под 45°. |
| 6 | Lvl 6 | **Крюк-кошка** | ПКМ | Крюк летит к маркеру или к партнёру. Игрок притягивается к цели. |
| 7 | Lvl 7 | **Парение** | Удерживать W/Space | Медленный спуск до 2 сек. Горизонтальный контроль сохраняется. |
| 8 | Lvl 8 | **Призыв платформы** | ПКМ стоя | Твёрдая платформа у курсора на 4 сек. Партнёр может вставать. Cooldown 6с. |
| 9 | Lvl 9 | **Обмен позициями** | Оба ПКМ → 3 сек | Оба нажимают ПКМ → отсчёт 3 сек → игроки телепортируются друг к другу. |
| 10 | Lvl 10 | **Заряженный дэш** | Удержать Shift | Зарядка 1.5 сек → рывок в 3× дальше в любом из 8 направлений. |

### Получение способностей
- В конце каждого уровня стоит **AbilityOrb** (светящийся шар)
- При подходе: подсказка "Нажми W/Space"
- При сборе: полноэкранная анимация с названием способности
- Финальная секция уровня непроходима без только что полученной способности

## 6. Кооперативные механики

### Базовые (Level 1)
- **Pressure Plate + Door** — один стоит на плите, дверь открыта, второй проходит
- **Буст-прыжок** — один приседает, второй прыгает от него на недосягаемую платформу
- **Синхронные кнопки** — оба нажимают одновременно (таймер 3 сек) → механизм срабатывает

### Продвинутые (Level 10)
- **Крюк партнёра** — крюком можно притянуть партнёра к себе
- **Призыв под партнёра** — вызвать платформу прямо под падающим партнёром
- **Обмен для загадок** — попасть в закрытую зону одним игроком, потом поменяться
- **Движущаяся платформа + рычаг** — один держит рычаг, второй запрыгивает

## 7. Уровни

### Level 1 — Подножие горы (полный)
**Визуал:** Ночное небо, скальные платформы, тусклые фонари, тёмно-синий фон  
**Структура:**
1. Туториал-зона: плашки "WASD — движение", "W/Space — прыжок"
2. Базовый платформинг (без способностей)
3. Кооп-головоломка: Pressure Plate + Door
4. AbilityOrb → полноэкранный анимированный текст "Дэш разблокирован"
5. Финальная секция: пропасть шире обычного прыжка — только дэш

### Level 10 — Вершина горы (полный)
**Визуал:** Облака, рассветное небо, кристаллические платформы  
**Структура:**
1. Секция дэша + заряженного дэша (Lvl 1 + Lvl 10)
2. Секция крюка + обмена позициями (Lvl 6 + Lvl 9)
3. Секция парения + призыва платформы (Lvl 7 + Lvl 8)
4. Финальный подъём: все механики комбинируются
5. Cutscene: оба персонажа на вершине, частицы конфетти

### Levels 2–9 — Заглушки
Одна комната, одна платформа, дверь выхода сразу открыта. Видны в Level Select как залоченные, но переключаются для демонстрации.

## 8. Сетевая архитектура

### Схема
```
Клиент 1 (Хост)          Node.js Брокер           Клиент 2 (Гость)
     │                        │                         │
     │──player:input──────────▶│                         │
     │                        │──game:state────────────▶│
     │◀───────────────game:state                        │
     │                        │◀────────player:input────│
     │──lobby:createRoom──────▶│                         │
     │                        │──lobby:list────────────▶│ (всем в лобби)
```

### WebSocket события

**Клиент → Сервер:**
- `player:input` — `{keys, mouse, seq, tick}`
- `lobby:createRoom` — `{name}` → сервер отвечает `{roomId}`
- `lobby:joinRoom` — `{roomId}`
- `lobby:renameRoom` — `{roomId, name}` (только хост)
- `lobby:leaveRoom` — `{roomId}`
- `game:ready` — `{levelId}`
- `ability:swapRequest` — `{}` (инициирует обмен позициями)
- `ability:swapConfirm` — `{}` (второй игрок подтверждает)

**Сервер → Клиент:**
- `game:state` — `{p1: {x,y,vx,vy,abilities}, p2: {...}, entities: [...], tick}`
- `lobby:list` — `[{id, name, level, playerCount, createdAt}]`
- `lobby:roomCreated` — `{roomId, name}`
- `lobby:roomDeleted` — `{roomId}`
- `player:joined` — `{role: "host"|"guest"}`
- `ability:unlocked` — `{ability, playerId}`
- `game:swapConfirm` — `{p1pos, p2pos}`

### Тиковая система
- Сервер: 20 тиков/сек (каждые 50 мс)
- Сервер авторитетный — физика Arcade считается на сервере (через headless Phaser или ручная математика)
- Клиент интерполирует позиции между тиками
- Input имеет sequence number для reconciliation

### Запуск — нет ничего для игроков

Игроки не запускают никаких команд и не вводят никаких IP.

**Архитектура:**
- Node.js брокер **задеплоен на Railway/Render** (бесплатный tier) с первого дня разработки
- URL брокера зашит в клиентском конфиге (`VITE_WS_URL` в `.env`)
- Игрок просто открывает браузер → клиент автоматически подключается к брокеру
- Один нажимает «Новая игра» → комната создаётся на брокере → появляется в списке у всех
- Второй видит комнату в списке → нажимает «Войти»

**Для разработки** (только разработчик, не игроки):
```bash
npm run dev   # запускает Vite (клиент) + Node.js (сервер) одновременно
```

**Для демо на защите:**
- Оба ноутбука открывают один URL (deployed или localhost если оба на одной машине для теста)
- Никакого ввода IP, никакого терминала у игроков

## 9. Лобби — UX-флоу

### Экран лобби
```
┌─────────────────────────┬──────────────────────────────┐
│   Доступные серверы     │      Выберите профиль        │
│                         │                              │
│ [Поиск по названию...]  │  ┌──────────────────────┐   │
│                         │  │ Мир123456            │   │
│ ┌─────────────────────┐ │  │ Уровень 1 — 0ч 12м   │ [Clear Save] │
│ │ Мир123456    Войти  │ │  └──────────────────────┘   │
│ │ 26.05.2026 · Lvl 1  │ │                              │
│ └─────────────────────┘ │  ┌──────────────────────┐   │
│                         │  │     Новая игра        │   │
│                         │  └──────────────────────┘   │
└─────────────────────────┴──────────────────────────────┘
```

### Правила
- **Пустой слот** → клик → создаёт комнату `Мир######` (случайные 6 цифр) → открывает Level Select
- **Занятый слот** → клик → переподключается к существующей комнате → открывает Level Select
- **Clear Save** → удаляет запись из localStorage, сервер удаляет комнату если она стала пустой
- **Список серверов** обновляется в реальном времени через WebSocket событие `lobby:list`
- **Поиск** фильтрует по названию локально (не запрос к серверу)
- **Сервер удаляется** автоматически когда последний игрок вышел → `lobby:roomDeleted` рассылается всем

### Level Select
- Вверху по центру: `[Название сервера] ✏️` — только хост может редактировать
- Редактирование инлайн: клик на карандашик → input field → Enter сохраняет, шлёт `lobby:renameRoom`
- Слева: аватар хоста (синий) + статус
- Справа: аватар гостя (оранжевый) или `?` если ещё не зашёл
- Кнопка "Начать" активна только когда оба игрока в лобби
- Только хост выбирает уровень
